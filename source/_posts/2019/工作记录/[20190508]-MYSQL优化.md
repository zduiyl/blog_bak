#### 偏移量查询优化


分页查询
SELECT * from user WHERE id >= (SELECT id from user ORDER BY id limit 1000,10)


的从 结果 集中 筛选出 前面的 结果  返回..

偏移量较大的 时候使用 速度较快,   因为 limit 1000,1 时  mysql需要把 指针从第一位 一直 挪动到 第一千位 ,然后从结果集中读取10跳

而使用子查询时由内部查询id通过主键 得到  速度较快,然后在将其作为筛选条件得到结果
